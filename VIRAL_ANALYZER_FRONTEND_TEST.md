1. **拖拽上传**

2. **支持格式**
   - MP4, MOV, AVI, MPEG
   - 最大 500MB

3. **预期效果：**
   - ✓ 上传成功提示："上传成功，AI 正在分析中..."
   - ✓ 列表自动刷新
   - ✓ 新上传的视频出现在列表顶部
   - ✓ 状态显示"分析中"（蓝色标签）

---

### **Step 3: 等待分析完成**

等待 **5-10 秒**，刷新页面

**预期效果：**
- ✓ 状态变为"已完成"（绿色标签）
- ✓ 视频卡片可点击

---

### **Step 4: 查看详情**

点击任意"已完成"的视频卡片

**预期效果：**

#### **左侧 - 视频播放器**
- ✓ 视频自动加载
- ✓ 可以播放/暂停
- ✓ 显示视频时长

#### **右侧 - 拆解报告**

1. **HOOK（前 3 秒）**
   - 紫色渐变卡片
   - 时间范围：00:00 — 00:03
   - 内容描述

2. **SELLING POINTS**
   - 绿色标题
   - 3-5 个卖点列表
   - 示例：40dB 降噪、14 小时续航...

3. **PACING**
   - 蓝色标题
   - 节奏分析描述
   - 示例：9 个分镜 / 30 秒，平均 3.3s 一镜

4. **STYLE**
   - 橙色标题
   - 风格分析描述
   - 示例：冷色调 + 局部金色暖光

5. **提取的创作因子**
   - 5 个维度：视觉风格、开场手法、旁白风格、节奏、行动号召
   - 网格布局展示

## 🐛 常见问题

### **问题 1：上传失败**
**症状：** 提示"上传失败"

**解决：**
1. 检查后端是否运行：`curl http://localhost:3000/api/health`
2. 检查文件大小是否 < 500MB
3. 检查文件格式是否为视频文件

### **问题 2：视频无法播放**
**症状：** 详情页黑屏

**解决：**
1. 检查浏览器是否支持视频格式
2. 打开开发者工具 → Network，查看视频请求是否成功
3. 确认后端视频流接口正常：`/api/viral-analyzer/videos/:id/stream`

### **问题 3：列表为空**
**症状：** 主页面显示"还没有拆解记录"

**解决：**
1. 上传一个新视频
2. 或检查后端数据库：`docker exec docker-postgres-1 psql -U vidcraft -d vidcraft -c "SELECT * FROM analyzed_videos;"`

### **问题 4：状态一直是"分析中"**
**症状：** 视频状态不更新

**解决：**
1. 刷新页面
2. 检查后端日志：`tail -50 /tmp/backend-viral.log`
3. 当前使用 Mock 数据，分析应在 5 秒内完成
