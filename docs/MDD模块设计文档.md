## M0 游客 / 演示模式

赛题原则要求「评委应能在最短路径内看到项目价值，外链尽量避免登录门槛」。

|  |
| --- |
| **DEMO-001 ● P0** #demo #guest #P0
**用户故事**
作为评委或访客，我希望无需注册即可进入 VidCraft 演示环境，直接体验完整的「商品解析→剧本生成→视频创作」流程，以便在最短路径内评估系统价值。

**验收标准**
* 演示环境首页提供「游客体验」按钮，点击后自动使用预置演示账号登录（无需输入账号密码）。
* 游客账号预置：已完成商品解析的示例项目 × 2、已生成剧本 × 3（含因子替换示例）、已生成视频 × 1（含数据看板 Mock 数据）。
* 游客账号不可修改密码、不可删除预置数据，但可以新建项目、上传素材、生成剧本、提交视频生成任务。
* 游客账号配额限制：每次会话最多生成 2 条视频，不消耗真实 API 配额（视频生成可使用 Mock 流程演示进度）。
* README 中明确说明：演示地址 + 游客入口位置 + 预置内容说明。 |

|  |
| --- |
| **DEMO-002 ● P0** #demo #preset #P0
**用户故事**
作为系统，我希望预置充分的演示数据，使评委能完整体验数据飞轮逻辑（生成→数据看板→AI诊断→优化建议），无需等待真实 AI 生成。

**验收标准**
* 预置演示视频包含 Mock 指标数据：完播率（2条：一条正常30%+，一条低于30%触发诊断提醒）、转化率、GMV。
* 低转化视频预置 AI 诊断报告，点击「AI 诊断」直接返回预置结果（无需真实调用大模型），演示诊断→优化建议→跳转分镜编辑的完整闭环。
* 预置爆款基因库 30 条示例（各品类），优质视频库 5 条示例（含拆解报告），保证检索功能可演示。 |

## M1 用户认证与账户

负责注册、登录、鉴权、Token 管理及个人信息维护。

v2.0 说明：M1 的 6 个 P0 Issue 仍然保留（生产环境需要），但演示路径通过 M0 游客模式绕过，评委无需走注册流程。

|  |
| --- |
| **AUTH-001 ● P0** #auth #register #P0
**用户故事**
作为一名新商家，我希望通过邮箱+密码完成账号注册，以便开始使用 VidCraft 创作视频。

**验收标准**
* 输入邮箱、密码（≥8位，含字母+数字）、确认密码，点击注册。
* 邮箱格式校验不通过时实时提示错误。
* 密码强度不足时提示具体要求。
* 邮箱已注册时提示「该邮箱已存在，去登录」。
* 注册成功后自动登录并跳转至首页（项目列表）。
* 注册成功写入 users 表，plan\_type 默认 free，video\_quota 默认 3。 |

|  |
| --- |
| **AUTH-002 ● P0** #auth #login #P0
**用户故事**
作为已注册商家，我希望通过邮箱+密码登录，获取访问令牌，以便操作我的项目与视频。

**验收标准**
* 输入邮箱+密码，点击登录。
* 密码错误时提示「邮箱或密码错误」。
* 登录成功后返回 JWT Access Token（7天有效）+ Refresh Token（30天）。
* Token 存储于 httpOnly Cookie。
* 登录后跳转至上次访问的页面（或默认项目列表）。 |

|  |
| --- |
| **AUTH-003 ● P0** #auth #token #P0  
**用户故事** 
作为已登录用户，我希望 Access Token 过期时系统能无感刷新，以便不必频繁重新登录。 
**验收标准**
* Access Token 过期（401 响应），前端拦截器自动使用 Refresh Token 换取新 Access Token。
* Refresh Token 也过期时，清除本地凭证并跳转至登录页，提示「登录已过期，请重新登录」。 
* 并发请求同时触发刷新时，只发起一次刷新请求，其余请求排队等候。 



|  |
| --- |
| **AUTH-004 ● P0** #auth #logout #P0
**用户故事**
作为已登录用户，我希望能够安全退出，以便在共享设备上保护账户安全。

**验收标准**
* 点击「退出」后清除本地 Token，Refresh Token 服务端加入黑名单（Redis TTL）。
* 退出后跳转至登录页。
* 退出后原 Token 无法再访问受保护接口。 |

|  |
| --- |
| **AUTH-005 ● P2** #auth #profile #P0
**用户故事**
作为已登录用户，我希望查看并修改我的昵称、头像，以便个性化账户信息。

**验收标准**
* 个人中心页展示：昵称、邮箱（不可修改）、当前套餐、本月剩余配额。
* 可修改昵称（1–30字）。
* 可上传头像（≤2MB，JPG/PNG，自动裁剪为 1:1）。 |

|  |
| --- |
| **AUTH-006 ● P2** #auth #quota #P0
**用户故事**
作为商家，我希望在任何页面都能直观看到本月剩余生成配额，以便合理安排视频生产计划。

**验收标准**
* 顶栏右侧展示「剩余配额：N 条」，配额 ≤ 2 时变橙色。
* 配额为 0 时生成按钮置灰，点击时弹窗提示升级套餐。
* 每次视频生成任务完成后自动扣减配额并刷新显示。 |

* 聚类结果由 Doubao 大模型归纳为结构化创作模板（策略+因子），写入 viral\_genes 表。
* 基因库更新后，导演 Agent 下次生成剧本时自动引用最新基因。
* 演示阶段：聚类逻辑可手动触发，展示「视频库→基因库」的完整提炼流程。 |
